Module Rules / Guidelines

* CMS modules follow same rules as content-modules.

Module example: CMS (with .content-container DIV)

integrating CMS is similar to other modules – everything contained within .content-container.
however, content container can either be plan DIV (as usual) with CMS Collection List Wrapper directly inside as a content-item. Or the top Collection List Wrapper container can have class of .content-container directly.
The first setup is handy if you want to add more content-items at the same level as CMS Collection List, eg. a button

the CMS Collection Item (.cms-item), is the wrapper for all item content (images, titles, paragraphs, etc). This is where the "display tile" is created.

classes used when CMS Collection list is inside of .content-container DIV:
.content-container (DIV)
.cms-container (CMS Collection List Wrapper)
.cms-grid (Collection List)
.cms-item (Collection Item)

combo classes for .cms-grid:
.cols-1111
.cols-2211
.cols-3211
.cols-3311
